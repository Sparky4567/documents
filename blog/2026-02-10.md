# Kodėl naudojant Cloudflare Page Builder reikia susikurti .jsonc?

> TLDR;

> Šiandienos tema: Kodėl naudojant Cloudflare Page Builder man reikia susikurti .jsonc?

Sveika, elektroerdve,

Taigi, taigi, taigi... Cloudflare Web Pages maloniai pasikeitė. Ir šis pokytis atnešė malonių akiai dizaino pokyčių bei pataisymų. Šie savo ruožtu kartu atsitempė  Web Building'o procesų pokyčių, kurie naudojantiems Page Builder paslaugą atnešė keliančią nuostabą problemą skambančią bene šitaip...

Em... O kur nurodyti  savo naudojamą build direktoriją... 

Tam ir skirtas šis įrašas. Jog trumpai apšviestų ir paaiškintų kaip išspręsti šią problemą be ilgų įžangų.

### wrangler.jsonc

Nuo šiol reikia sukurti vieną papildomą failą ! :nerd_face:

```
touch wrangler.jsonc
nano wrangler.jsonc

{
    "name": "documents",
    "compatibility_date": "2026-02-09",
    "assets": {
    "directory": "./build"
  	}
}

ctrl+o
Enter
ctrl+x
```

```
name:___
```

Projekto vardas turintis sutapti su vardu naudojamu Pages paslaugos grafinėje sąsajoje.

```
assets:{directory: ______ }?
```

Sugeneruotų statinių failų direktorijos vardas, kuris nurodo iš kur turi būti paimti failai replikavimui bei hostingui.

Papildomi kintamieji (pvz. GIT_USERNAME) nurodomi nuostatose naudojant grafinį vedlį arba projekto nuostatose.

Šiam kartui tiek.

Iki sekančio susiskaitymo.

Šis ekranas trumpam išsijungia, bet kažkur įsijungia kitas.
