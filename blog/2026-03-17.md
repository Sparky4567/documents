Šiame įraše dalinuosi markdown įrašų sinchronizacijos metodu, kuris leidžia pernaudoti tinklaraščio markdown įrašus tuo atveju, jei naudojama ChromeOS bei konteinerizuota Linux aplinka (versija: crostini/bookworm).

### Google Drive direktorijos pasidalinimas su virtualiu Linux konteineriu

Atsidarome failų naršyklę, kairiojoje pusėje pasirenkame Google Drive, pasižiūrime į direktorijų sąrašą, dešiniuoju pelės klavišu paspaudžiame direktoriją, kuria norime pasidalinti su Linux aplinka ir kairiuoju klavišu pasirenkame "Share with Linux".

Ypatingai svarbu atkreipti dėmesį į direktorijos pavadinimą, kadangi nors ir direktorijos pavadinimas išliks tas pats, bet šiosios naudojamas kelias Linux aplinkoje bus sudarytas iš:

- Pagrindinio kelio /mnt/chromeos/GoogleDrive/
- Pridėtinės jo dalies (Ši atitiks Google Drive naudojamos direktorijos pavadinimą, pvz:DOCS_STORAGE)

Tad visas naudojamas kelias bus: /mnt/chromeos/GoogleDrive/MyDrive/DOCS_STORAGE



### Programinis kodas

Už talpinimą atsakinga programinė dalis labai paprasta, tačiau deramam jos veikimui būtina įsidiegti astral paketų tvarkyklę. 

- Pagrindinėje projekto direktorijoje susikuriame naują. Ją pavadiname "python_scripts_portable".
- Terminalo pagalba inicializuojame naują uv projektą - uv init
- uv add pathlib
- Python skriptų direktorijoje sukuriame naują failą "movetodrive.py"
- Į šį įklijuojame žemiau pateiktą Python kodą
- Paredaguojame kodą taip, kad atitiktų asmeninio projekto reikmes

***Atkreipti dėmesį į paieškos direktoriją get_md_files() funkcijoje , kadangi maniškė pavadinta blog, o jūsų gali vadintis visai kitaip***

 ***Taip pat atkreipti dėmesį į file_name kintamojo reikšmę, kurioje panaudotas su Linux bendrinamas Google Drive kelias***

```python
from pathlib import Path
from pathlib import PurePath
# import random
import shutil
def get_md_files(passed_dir):
    files = Path(passed_dir).rglob("*.md")
    return files

def get_list_to_copy():
    files = get_md_files("../blog")
    path_list = []
    for file in files:
        path_list.append({"file_name":file.name,"file_path":file.resolve().as_posix()})
    return path_list

def file_copy_function():
    try:        
        files_from_astro = get_list_to_copy()
        for ob in files_from_astro:
            file_name = Path(f"/mnt/chromeos/GoogleDrive/MyDrive/DOCS_STORAGE/{ob["file_name"]}").resolve()
            if(not file_name.is_file()):
                print(f"{file_name} does not exist. ❌")
                shutil.copy(ob["file_path"],file_name)
                print(f"{file_name} was copied to {file_name}. ✅")
            else:
                print(f"{file_name} does exist. Skipping. ✅")


    except Exception as e:
        print(f"Error: {e}")

file_copy_function()
```

**Bash skriptas patogesniam failų perkėlimui bei alias sukūrimui (talpinamas pagrindinėje projekto direktorijoje)**

```bash
#!/bin/bash
cd python_scripts_portable && uv run movetodrive.py
```

**alias - aprašomas ~/.bashrc faile, terminale naudojamas, kaip komandos trumpinys**

```bash
alias upload-doc='cd /home/my-user/github/documents && ./movetodrive.sh'
```

### Failų perkėla

terminale suvedame greitąją komandą upload-doc, spaudžiame Enter klavišą ir palaukiame, kol baigsis Python skripto pagalba vykdoma failų perkeltis.

### Naudojimas su Gemini

Po perkėlos įsitikinti, jog failai sinchronizuoti - atsidaryti naršyklę naršyklę bei naudotą Google Drive direktoriją perkėlimui.

Įsitikinus, jog failai atvaizduojami ir juos galima atverti -> Aplankyti Gemini. 

Įrašų naudojimui pasirinkti Plius mygtuką (Add files) -> Google Drive -> Add from Drive -> Pasirinkti direktoriją, kurioje turėtų būti failai.
